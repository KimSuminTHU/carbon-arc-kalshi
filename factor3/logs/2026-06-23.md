# Factor 3 실험 로그 — 2026-06-23 · LLM 점수 taxonomy first-principles 확정

> 산출: `DESIGN.md §2`(개정), `PLAN.md §3`·`RESULTS.md`·`README.md` 정합성 갱신 · 관련: 2026-06-22 킥오프

## 목적
스크립트 작성 전, **LLM이 뱉을 점수(Option A/C)와 예측(B)을 무엇으로 정의할지**를 임의 목록이 아니라 원리에서 유도해 확정. (코드 실행은 다음 로그.)

## 결정 (rationale)
- **출발점:** Y = 매출 서프라이즈 = (actual − consensus)/consensus = **잔차**. ⇒ 예측가능 부분은 단 하나의 인과식:
  **예측가능 서프라이즈 = (컨센서스에 아직 안 들어간 매출-방향 신호) × (신호 신뢰도).**
- 점수는 이 사슬의 **항**이어야 함:
  - **S1 RevVsConsensus** (−100..+100) — SIGNAL(방향), 헤드라인.
  - **S2 NewsNotInConsensus** (0..100) — NOVELTY(미반영분만 잔차 예측).
  - **S3 SignalReliability** (0..100) — RELIABILITY(구체성·수치화·track record·헤징).
  - **S4 CardNarrativeConsistency** (−100..+100) — C 전용 교차검증(카드↔발언).
- **`MarginCostRisk` 삭제**: 매출=top-line(가격×수량); 비용·마진은 **EPS** 항. repo상 비용→EPS null → 매출 모델에 넣으면 **카테고리 오류 + 자유도 낭비**. (Y=EPS 확장 시 부활.)
- **절대 수요 점수(DemandStrength/Momentum/ForwardRevenueDirection) → S1로 흡수**(중복 + *절대*는 *잔차* 예측에 약함). **confidence/specificity → S3 통합**.
- **consensus anchor**를 프롬프트에 제공(공개·print 이전 값) = leakage 아님, 서프라이즈 질문의 필수 baseline.
- **n≈105 → 검약이 최적**: "풍부하게 추출, 검약하게 회귀" — 보조 점수(pricing/expansion/guidance-vs-prior)는 추출만, 헤드라인 OLS엔 S1–S3(+S4)만.

## A/B/C 정의 (확정)
- **A** = LLM이 Z만 → 점수 → **OLS**가 예측. **B** = LLM이 입력→숫자 직접. **C** = LLM이 X+Z 융합→피처 → OLS.

## Verdict
점수 taxonomy를 원리 기반으로 확정. "최선"은 주장 대신 **실험이 판정**(반증가능):
A1(S1만) vs A3(S1+S2+S3) · consensus-relative vs absolute · A/C vs B(taxonomy 가치) — 셋 다 `DESIGN.md §7`·`RESULTS.md`에 ablation으로 등재.

## Caveats
- 점수 정의가 합리적이어도 LLM이 일관되게 매길지는 스모크에서 확인 필요(분산·거부율).
- consensus anchor 제공이 anchoring 편향을 유발할 수 있음 → 절대 arm과 비교로 점검.
